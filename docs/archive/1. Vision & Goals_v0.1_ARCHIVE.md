## 1. Vision & Goals

> **"A single screen where a 5-year-old and a grandparent can play charades together — no reading required."**

Kid-Friendly Charades is a free, open-source, mobile-first web app that lets families play charades across age groups seamlessly. Players aged 7 and under use **Picture Charades** (image-based prompts); players aged 8 and older use **Word Charades** (text-based prompts). Both modes share the same game flow, scoring, and UI — so nobody gets left out.

| Goal | Success Metric |
|------|----------------|
| Enable mixed-age families (adults + kids) to play charades in a single UI | > 500 unique sessions/week (GitHub Pages analytics) |
| Support two age-appropriate play modes | 80% of sessions use the mode matched to the selected age group |
| Mobile-first, accessible, responsive UI | Lighthouse mobile score ≥ 90 |
| Single-click deploy via GitHub Pages | CI/CD working end-to-end via GitHub Actions |
| Core MVP built within 5 hours of dev time | Time-tracked ≤ 5 h |

## 2. User Personas

### 🧒 Young Player (Age 4–7)
- Cannot read fluently; relies on pictures and icons
- Needs large, colorful visuals and simple UI cues
- May be guided by a parent to interact with the screen

### 👦 Older Child / Tween (Age 8–12)
- Comfortable reading short words
- Enjoys a challenge; appreciates category variety (animals, movies, sports, etc.)
- Likely the one holding the phone

### 👨‍👩‍👧 Adult / Parent
- Sets up the game and manages scoring
- Wants minimal friction to get the game started
- May play against or alongside kids

### 👴 Grandparent / Extended Family
- Needs large text and clear navigation
- Unfamiliar with complex UI patterns; relies on intuitive flow

## 3. Core Features (MVP)

### 3.1 Home / Lobby

**Description:** Landing screen where the game is configured before it starts.

**Includes:**
- App name/logo and tagline
- Game Mode: **Picture Charades (Under 7)** | **Word Charades (8 and up)** | (future upgrades) **Who Am I (8 and up)**
- Optional team name inputs (Team A / Team B, customizable)
- Category multi-select (e.g., Animals, Food, Sports, Movies, Everyday Objects)
- Game style: Default (click and go) | "Heads Up" style (tilting phone right/pass with timer each round)
- "Start Game" CTA button

**Owner:** UI/UX Designer + Front-end Engineer
**Estimated Effort:** 30 min

---

### 3.2 Mode Switch

**Description:** On game start, the app automatically loads the appropriate content pool (images vs. words) based on the age mode selected in the lobby.

**Logic:**
- Under 7 → load `pictures.json`, display image card
- 8 and up → load `words.json`, display text card

**Owner:** Front-end Engineer
**Estimated Effort:** 15 min

---

### 3.3 Picture Charades Mode

**Description:** Full-screen image prompt for young players.

**Includes:**
- Random image drawn from curated set (≈ 30 items, categorized)
- Image displayed fullscreen or in a large centered card
- Image caption shown alongside the picture, even though the actor doesn't need to read it to play — "no reading required" means literacy isn't a barrier to *playing*, not that words are hidden. Seeing the word next to its picture is intentional incidental literacy exposure for pre-readers.
- 30-second countdown timer (prominent, visual)
- **"Got It ✓"** and **"Pass ✗"** buttons
- **"Next"** button after round resolves

**Owner:** Front-end Engineer
**Estimated Effort:** 45 min

---

### 3.4 Word Charades Mode

**Description:** Large-font text prompt for older players and adults.

**Includes:**
- Random word drawn from curated list (≈ 50 items, categorized)
- Word displayed in large, bold, readable font
- Same 30-second timer and Got It / Pass controls as Picture mode

**Owner:** Front-end Engineer
**Estimated Effort:** 30 min

---

### 3.5 Scoring

**Description:** Simple tally tracking across teams within a session.

**Includes:**
- Per-team score increments on "Got It ✓" tap
- Scoreboard visible between rounds (or accessible via a persistent score icon)
- Scores persisted in `localStorage` (survive accidental page refresh)
- Optional end-of-game summary screen with winner announcement

**Owner:** Front-end Engineer
**Estimated Effort:** 20 min

---

### 3.6 Responsive Layout

**Description:** Mobile-first layout that scales gracefully to tablets and desktops.

**Requirements:**
- Base design at 375px width (iPhone SE)
- Breakpoints at 768px (tablet) and 1024px+ (desktop)
- CSS Grid and Flexbox; no external CSS frameworks
- Touch-friendly tap targets (≥ 44×44px per WCAG)

**Owner:** UI/UX Designer
**Estimated Effort:** 30 min

---

### 3.7 GitHub Pages CI/CD

**Description:** Automated deployment pipeline so the app is always live with one push.

**Workflow:** `lint (ESLint) → build (copy/bundle) → deploy to gh-pages branch`

**Owner:** DevOps Engineer
**Estimated Effort:** 20 min

---

### 3.8 Accessibility

**Description:** Baseline a11y so the app is usable without special effort.

**Includes:**
- Alt text on all images (short, descriptive)
- High-contrast color palette (WCAG AA minimum)
- Keyboard navigation for all interactive elements
- ARIA labels on icon-only buttons
- Focus indicators visible on all interactive elements

**Owner:** UI/UX Designer
**Estimated Effort:** 15 min

---

### 3.9 Documentation

**Description:** README and contribution guide for open-source handoff.

**Includes:**
- Project overview and screenshots
- Local development setup (clone → serve)
- Deployment steps (GitHub Pages, manual and CI)
- How to add new words/images (content contribution guide)

**Owner:** Technical Writer
**Estimated Effort:** 15 min

## 4. Non-Functional Requirements

| Category | Requirement |
|----------|-------------|
| **Performance** | Page load < 1 s on simulated 3G (Lighthouse); images optimized (WebP/compressed PNG, max 100 KB each) |
| **Security** | No external API calls; all assets static; no user data transmitted off-site |
| **Privacy** | No personal data collected; no cookies beyond session state; optional self-hosted analytics only |
| **Maintainability** | Pure HTML + CSS + vanilla JS (ES6 modules); no heavy frameworks; codebase readable by a junior dev |
| **Scalability** | Fully static; GitHub Pages CDN handles any traffic spike |
| **Offline Support** | Service Worker (Workbox minimal config) enables play without internet after first load |
| **Internationalization** | All UI strings externalized in `strings.json` (ready for i18n in a future release) |
| **Browser Support** | Latest 2 versions of Chrome, Safari, Firefox, and Samsung Internet (covers 95%+ of mobile users) |

## 5. Tech Stack

| Layer | Technology | Rationale |
|-------|------------|-----------|
| **Hosting** | GitHub Pages (`gh-pages` branch) | Free, zero-config static hosting |
| **Build** | `npm` scripts (no bundler for MVP) | Keeps build trivial and fast |
| **UI** | HTML5, CSS3 (variables, Flex, Grid), vanilla JS (ES6 modules) | No dependencies = no maintenance debt |
| **Images** | SVG / optimized PNG in `/assets/images/` | Committed to repo; no external CDN needed |
| **Content** | `assets/data/pictures.json`, `assets/data/words.json` | Easy to edit and extend without touching code |
| **State** | `localStorage` | Scores and settings survive page refresh; no backend needed |
| **Offline** | Service Worker via Workbox (minimal config) | Offline-first for family game nights |
| **CI/CD** | GitHub Actions (`node:18` runner) | Auto-deploy on push to `main` |
| **Linting** | ESLint (flat config) | Consistent code quality |

## 6. Information Architecture & User Flows

### App Screens

```
Home / Lobby
  └─ [Start Game]
       └─ Round Screen
            ├─ Picture Charades (if age < 8)
            └─ Word Charades (if age ≥ 8)
                 ├─ [Got It ✓] → increment score → next card
                 ├─ [Pass ✗] → no score → next card
                 └─ [Timer Expires] → next card
                      └─ End-of-Round Score Summary
                           └─ [Play Again] / [End Game]
```

### Key Interactions

| Trigger | Action |
|---------|--------|
| Select age mode + press Start | Load appropriate content pool and enter Round Screen |
| Tap card | No action (prevents accidental reveals); actor holds phone away |
| Timer hits 0 | Auto-advance to result (no score awarded) |
| Got It ✓ | +1 to active team score; brief success animation; next card |
| Pass ✗ | 0 points; brief neutral animation; next card |
| All cards exhausted | Shuffle and restart pool (infinite play) |

## 7. Content Strategy

### Picture Charades (≤ 7 y) — ≈ 30 items
- Simple, unambiguous illustrations (single subject, no text)
- Categories: Animals, Food & Drinks, Vehicles, Everyday Objects, Nature
- Visual style: flat, colorful, high-contrast (consistent art style across all cards)
- Source: curated SVG illustrations (open-license) or original art
- Stored in: `assets/images/picture-charades/`
- Metadata in: `assets/data/pictures.json` (id, filename, label, category, ageMin, ageMax)

### Word Charades (≥ 8 y) — ≈ 50 items
- Common nouns, verbs, and pop-culture references appropriate for families
- Categories: Animals, Movies & TV, Sports, Food, Famous People (fictional OK), Actions
- Stored in: `assets/data/words.json` (id, word, category, ageMin, difficulty)

### Category Selection (Lobby)
- Players can multi-select categories; only matching cards are loaded into the round pool
- Default: all categories selected
