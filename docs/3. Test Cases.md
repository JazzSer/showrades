## SR-050.6 — Regression Matrix for Round Types

Reference: `docs/2. Backlog & Status.md` SR-050.1–.5 (all fixed 2026-06-14, commit `2ab2b68`).

### Setup reference
- **Turn mode**: Setup screen → "By card" / "By round"
- **Goal mode**: Setup screen → "Endless" / "Points target" (set win target low, e.g. 5, for faster testing)
- **Timer**: Categories screen → Round Timer "0 / 30 / 60 / 90" (0 = off)
- **Category count**: pick **1 category** (~16-18 cards) for Endless combos so the deck exhausts quickly

### Checklist per combination

For each TC, run through and confirm:

**All combos:**
- [ ] Pass/transition screen appears before every card/turn, with mode-appropriate copy (card mode: role-swap wording; round mode: "give the phone to {team}")
- [ ] `roundend` screen only appears in **round mode** after a full team rotation — never in card mode
- [ ] `gameover` only appears at win target (Points) or deck exhaustion by the last team (Endless)

**Timer combos (TC-02, TC-04, TC-06, TC-08):**
- [ ] Timer resets to configured length and counts down on every card/turn, not just the first
- [ ] On expiry: "Time's up!" feedback shows, then auto-advances correctly (card mode: Pass-equivalent + team rotates; round mode: ends turn / passes to next team)

**Score attribution (card mode, TC-01–04):**
- [ ] On "Got It", the **guessing team** (not the team holding the phone) gets +1 — verify across multiple alternations

**Deck exhaustion (Endless, TC-03/04/07/08):**
- Card mode (TC-03/04): deck exhaustion ends the game (`gameover`) regardless of whose turn it is
- Round mode (TC-07/08): deck exhaustion mid-rotation (not last team) shows "Deck complete! ... pass the phone to the next team" with a "Pass to Next Team →" button — NOT "Finish Game"; only the **last team's** exhaustion triggers `gameover`/summary

**Win condition (Points, TC-01/02/05/06):**
- [ ] `gameover` triggers the moment any team reaches the win target — correct team, no overshoot
- [ ] Deck reshuffles seamlessly if exhausted before win target reached (game continues, no premature ending)
