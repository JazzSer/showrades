## 2026-06-11 — P0 Build Decisions Log

A working prototype (`docs/designs/Showrades Prototype.html`) was found to already implement a full game loop with rules that diverged from this backlog/PRD. Per product owner decision, **this backlog/PRD remains the source of truth** where they conflict, but the prototype's game rules were reviewed and explicitly locked in below. The following decisions resolve the P0 open items and were used to build the current `app/`, `hooks/`, and `lib/` code:

- **Architecture:** Next.js multi-route App Router (`/`, `/setup`, `/categories`, `/game`, `/game/summary`, `/game/end`) with a shared `GameProvider` React Context (`hooks/useGameState.tsx`) persisted to `localStorage` (`showrades_game_v1`) — chosen over a single-page state machine for cleaner separation of screens and easier future extension.
- **Reveal mechanic (Q3):** No hide/flip step. The actor sees the word/emoji directly and holds the phone away from teammates. Resolves "Decide on reveal mechanic."
- **Timer (Q2):** Configurable per round — 30 / 60 / 90 s, selected on the Categories screen. Resolves "Decide timer duration."
- **Categories (Q4 / "Finalise category list"):** Locked at 6 — Animals (mint), Food (sun), Fruits (coral), Jobs (sky), Veggies (plum), Sports (sky). Supersedes the original 5-category MVP list (Occupations → Jobs, added Sports).
- **Picture Charades visuals:** Emoji-based for now (no SVG illustration pipeline). Each word card carries an `emoji` field; Picture mode renders large emoji + word. Custom illustrations remain a future option.
- **Game rules locked in:** Win at 15 points (`WIN_TARGET`), "Got It" = +1, "Pass" = 0, multi-round play with shuffled/reshuffling deck, 2–6 teams supported.
- **"Heads Up" style (P2 item):** Implemented now as a label-only toggle ("HEADS UP" vs "YOUR TURN") — no device-tilt/DeviceOrientation logic. The full tilt-based interaction remains in P2 if desired.
- **Category selection (Q8):** Per-game (selected once on the Categories screen before starting), not per-player.

**Known issue fixed:** `app/game/page.tsx` had an edge case where navigating to `/game` with a fresh/default game state (`screen: "home"`) could leave the user stuck on the "pass the phone" screen after tapping "ready". Fixed by simplifying the play/pass screen condition.

**Outstanding from this build:** Interactive browser verification (timer countdown, button taps, multi-round flow) has not been performed — no browser automation tooling is available in the dev environment. All routes verified to return HTTP 200 with expected content via SSR, and `npx tsc --noEmit` passes with zero errors. Recommend a manual playtest pass on `http://localhost:3100` (or your dev port) covering: Setup → Categories → full round → Round Summary → Next Round → Game Over → Play Again.
